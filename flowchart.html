⚡ Triggers how it starts
⌨️ Manual (CLI)
User runs commands directly in terminal.
npm run start process -- --exam jeemain --year 2025 --shift 1
npm run scrape -- --exam neet --year 2024 --shifts 2
npm run test-models
npm run stats
Each stage is a separate command — user controls sequencing.
🤖 AI Agent (autonomous)
An LLM agent (like me) executes the full pipeline via shell commands.
Standard loop:
1. User says "process this exam"
2. Agent runs each stage sequentially
3. Agent reads errors & self-heals
4. Agent presents each question for human review
5. Agent runs final verification
See MASTER-PROMPT.md for the complete agent instruction set.
📐 Pipeline 9 stagesPDF → paper.json → subject files
0 Setup
one-time cd C:\QUESTION-PIPELINE npm install Set .env with API keys npx tsc --noEmit → Verify all 7 keys present + TypeScript compiles
1 Scrape
auto npm run scrape -- --exam <name> --year <Y> --shifts <N> or npx tsx src/scrapers/nta-scraper.ts → data/{exam}/raw/{exam}-{year}-{shift}.pdf fallback: gateoverflow-scraper.ts
2 OCR
auto Mistral OCR API 60 req/min npx tsx src/extractors/ocr-stage.ts --input <pdf> --output <dir> Chunks PDFs >3.5MB → base64 → Mistral → PageContent[] + base64 images → data/{exam}/ocr/{file}.json
3 Structure
auto AI Provider Chain extracts Question[] + Passage[] from OCR markdown npx tsx src/extractors/structurer.ts --input <ocr.json> --output <dir>
▶ #1 NVIDIA Qwen3 Coder 480B · 40 RPM · 262K ctx · ~900ms PRIMARY
▼ #2 LongCat Flash Lite · 30 RPM · 256K output · best for large papers BEST 4 BIG
▼ #3 Poolside Laguna M.1 · 30 RPM · 131K ctx · thinking disabled FALLBACK
▼ #4 Vanchin KAT-Coder · 20 RPM FALLBACK
▼ #5 Gemini 2.5 Flash · 5 RPM SLOW
▼ #6 Cerebras GPT-OSS-120B · 5 RPM · ≤12 pages only LAST
⚙️ Anti-hallucination: scans OCR for answer key → if absent, sets ALL answers to ""
⚙️ Fallback: if provider returns 0 questions or parse failure → try next in chain
⚙️ Rate limits: each provider has per-minute Bottleneck limiter (see structurer.ts:22-27)
4 Cache Diagrams
auto npx tsx src/extractors/diagram-cacher.ts --input <ocr.json> Tier 1: Mistral image coords → crop. Tier 2: sharp full-page crop → data/{exam}/diagrams/{subject}/q{3digit}-fig{n}.png
5 Validate
auto npx tsx src/validators/auto-validator.ts --path data/{exam}/{year}/{shift}/ 32 checks: IDs, types, options, diagrams, passages, HTML injection, duplicates... → Validation report { passed, failed, warnings }
6 Finalize
auto npx tsx src/finalizers/exporter.ts --exam <name> --year <Y> --shift <N>
1. normalizeTexts() — LaTeX→Unicode, fix OCR ligatures 2. normalizeTopics() — controlled vocabulary, fallback general-{subject} 3. assignIds() — {exam}-{year}-{shift}-{subject}-{3digit} 4. SHA-256 checksum (self-referential safe order)
📄 Outputs:  data/{exam}/{year}/{shift}/paper.json — all questions, full schema v4  data/{exam}/{year}/{shift}/{subject}.json — per-subject split
data/index.json — master index of all datasets
7 Human Review
human npm run review -- --exam <name> --year <Y> --shift <N> Interactive CLI: a=accept, e=edit, s=skip, f=flag, j/k=prev/next, q=quit Progress auto-saved to .review-progress.json AI agent fallback: present each question to user manually
8 Sign Off
human npm run signoff -- --exam <name> --year <Y> --shift <N> --status verified Writes review metadata → data/{exam}/{year}/{shift}/metadata.json
9 Verify
auto npm run verify SHA-256 checksum verification on ALL paper.json in data/ npm run rebuild-index Regenerate master index npm run stats Print dataset statistics

🔄 Error Recovery self-healing
Provider failures
NVIDIA timeout → try LongCat (next in chain)
LongCat 500 → check URL/API key, fall to Poolside
Gemini 401 → check API key + query param auth
All providers fail → abort, report to user
Data failures
OCR returns 0 pages → retry PDF with different chunking
AI returns 0 questions → auto-try next provider
JSON parse failure → re-prompt with stricter instructions
PDF not found → check filename, try fallback scraper
Validation fails → read specific error, fix, re-run
🧠 AI Agent Runbook for LLMs
When a user tells an AI agent to "process this exam paper":
  1. Install — cd C:\QUESTION-PIPELINE; npm install; npx tsc --noEmit
  2. Verify — npm run test-models to confirm all 6 AI providers respond
  3. Scrape — download PDF via npm run scrape -- --exam <name> --year <Y> --shifts <N>
  4. OCR — npx tsx src/extractors/ocr-stage.ts --input data/<exam>/raw/<file>.pdf --output data/<exam>/ocr/
  5. Extract — npx tsx src/extractors/structurer.ts --input data/<exam>/ocr/<file>.json --output data/<exam>/extracted/
  6. Validate — npx tsx src/validators/auto-validator.ts --path data/<exam>/<year>/<shift>/
  7. Finalize — npx tsx src/finalizers/exporter.ts --exam <name> --year <Y> --shift <N>
  8. Review — present each question to user: accept / edit / skip / flag
  9. Sign off — npm run signoff -- --exam <name> --year <Y> --shift <N> --status verified
  10. Final verif — npm run verify; npm run rebuild-index; npm run stats
⚡ Cross-validation shortcut (optional): run TWO providers on same OCR output, diff results, show only divergences (~80% less review time).
